## Описание программы


Weather Diary — графическое приложение на Python для учёта и анализа погодных данных. Позволяет:
* добавлять записи о погоде с указанием даты, температуры, описания и наличия осадков;
* просматривать все сохранённые записи в табличном виде;
* фильтровать записи по дате и температуре;
* сохранять данные в JSON‑файл и загружать их при следующем запуске.

## Требования

* Python 3.6 или выше;
* стандартные библиотеки Python: `tkinter`, `json`, `datetime`.

## Функциональность
## Приложение содержит следующие элементы:
## Поля ввода:
* Дата — в формате ДД.ММ.ГГГГ (например, 15.04.2026);
* Температура — числовое значение в градусах Цельсия;
* Описание погоды — текстовое поле (не может быть пустым);
* Осадки — переключатель «Да»/«Нет».

## Кнопки:

* «Добавить запись» — добавляет новую запись в таблицу;
* «Применить фильтры» — фильтрует записи по заданным критериям;
* «Сохранить в JSON» — сохраняет все записи в файл weather_data.json;
* «Загрузить из JSON» — загружает записи из файла weather_data.json.

## Фильтры:
* Фильтр по дате — показывает записи только за указанную дату;
* Фильтр по температуре — показывает записи с температурой выше указанного значения.

## Примеры использования
## Пример 1: добавление новой записи
* Введите дату в формате ДД.ММ.ГГГГ.
* Укажите температуру (например, 23.5).
* Напишите описание погоды (например, «Солнечно, безветренно»).
* Выберите «Да» или «Нет» для осадков.
* Нажмите кнопку «Добавить запись».

## Пример 2: фильтрация данных
* В поле «Фильтр по дате» введите дату (например, 15.04.2026).
* В поле «Фильтр по температуре (>) введите значение (например, 15).
* Нажмите «Применить фильтры» — в таблице останутся только записи за 15.04.2026 с температурой выше 15 °C.

## Пример 3: сохранение и загрузка данных
* Нажмите «Сохранить в JSON» — данные сохранятся в файл weather_data.json.
* Закройте и перезапустите приложение.
* Нажмите «Загрузить из JSON» — все ранее сохранённые записи появятся в таблице.

## Валидация ввода
* Приложение проверяет корректность данных:
* Дата должна быть в формате ДД.ММ.ГГГГ — иначе появится сообщение об ошибке.
* Температура должна быть числом — иначе появится сообщение об ошибке.
* Описание не может быть пустым — иначе появится сообщение об ошибке.
